# E2E Evidence — #68 Production docs (reference + gallery + migration)

- **Date**: 2026-06-16
- **Test**: `tests/test_docs.py`

## What was delivered

- `scripts/gen_tool_reference.py` — introspects the **live** FastMCP server and
  generates `docs/tools/reference.md` covering all tools (name, description,
  parameter table). Run it after any tool change.
- `docs/tools/reference.md` — **24 tools** documented (auto-generated).
- `docs/migration-0.x-to-1.0.md` — install, the 1.0 SemVer stability guarantee,
  what's new since 0.10 (Trust & Guard, plugins, remote render), deprecations.
- `docs/gallery.md` — domain gallery: **CFD / FEA / Medical / Geo / Molecular**,
  each with formats, fields, tools, and a starting workflow.

## Drift guard (the docs can't go stale)

`tests/test_docs.py` enforces the docs against the code:

```
5 passed
  test_render_markdown structure / empty params
  test_reference_covers_all_live_tools   -> 24 live tools all present in reference.md
  test_migration_guide_exists
  test_domain_gallery_exists             -> contains CFD / FEA / Medical
```

`test_reference_covers_all_live_tools` lists the live MCP tools and fails if any
is missing from `reference.md` — so adding a tool without regenerating the
reference breaks CI. Regenerating is idempotent (re-running the generator
produces no diff).
